**1. isTargetPage() 检测函数：**
```javascript
function isTargetPage() {
  return window.location.href.includes(TARGET_PATH);
}
```
使用 href.includes 检测，因为目标页面使用 hash 路由（`#/training/examination/exam-data`）。

**2. init() 函数实现：**
```javascript
function init() {
  if (!isTargetPage()) {
    return;
  }
  console.log(`[${SCRIPT_NAME}] 目标页面已激活，开始初始化`);
  // Phase 2-4 的实际处理逻辑将在此处添加

```

**4. SPA 路由检测（INIT-03，per D-03 纯 onurlchange）：**
```javascript
// SPA 路由切换检测（per D-03: 纯 window.onurlchange，无降级方案）
if (typeof window.onurlchange === 'object' && window.onurlchange === null) {
  window.addEventListener('urlchange', (event) => {
    console.log(`[${SCRIPT_NAME}] 检测到路由变化: ${event.url}`);
    init();
  });
}
```

关键实现细节：
- 检测条件 `window.onurlchange === null`（Tampermonkey 特征：支持时该值为 null 对象，不支持时为 undefined）— 这是 Tampermonkey 文档明确说明的检测方式
- 事件名是 `urlchange`（不是 `onurlchange`）
- 无 hashchange/popstate 降级方案（per D-03 KISS 原则）
- 路由变化时输出新 URL（per D-02 简洁日志：关键节点）
